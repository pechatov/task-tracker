# Task Tracker

Приложение помогает планировать личные задачи и календарные события на день, неделю и месяц. Контекст описывает язык планирования, а не техническую реализацию.

## Language

**Пользователь**:
Человек, для которого ведется планирование. Задачи, стримы, проекты и подключенные календари принадлежат конкретному пользователю.
_Avoid_: аккаунт, владелец

**Задача**:
Планируемое действие, которое пользователь создает сам и может завершить, перенести или отложить. У задачи есть дата выполнения, приоритет дня и состояние; описание, стрим и проект необязательны.
_Avoid_: тикет, событие, встреча

**Описание задачи**:
Необязательный текстовый контекст конкретной задачи: детали, ссылки, критерии выполнения или короткий чек-лист. Описание задачи не является самостоятельной заметкой.
_Avoid_: заметка, документ

**Задача на дату**:
Задача, назначенная на день без конкретного времени начала и окончания. Это основной способ планирования задач в первом релизе.
_Avoid_: событие на весь день

**Рутина**:
Повторяемая практика или привычка, которую пользователь отслеживает отдельно от обычных задач. Рутина не является задачей, хотя в будущем может создавать отметки выполнения или задачи на конкретные дни.
_Avoid_: повторяющаяся задача

**Временной блок**:
Конкретный интервал времени, который пользователь резервирует под задачу. Временной блок превращает задачу из простой задачи на дату в задачу, занимающую место в календарной сетке.
_Avoid_: встреча, календарное событие

**Календарное событие**:
Запись во внешнем или внутреннем календаре с временным интервалом. Событие отображается в плане дня/недели/месяца, но не является задачей.
_Avoid_: задача, дедлайн

**Ссылка события**:
URL, связанный с календарным событием, который пользователь открывает в браузере для просмотра деталей или перехода к встрече. Ссылка события может быть взята из места события, если там указан URL, и не является описанием события.
_Avoid_: описание встречи, тело события

**Подключенный календарь**:
Календарь из внешнего источника, который пользователь выбрал для отображения в планах. Один источник может давать несколько подключенных календарей; цвет подключенного календаря используется для маркировки его событий.
_Avoid_: аккаунт, интеграция

**Встреча**:
Календарное событие с участниками. Встреча является частным случаем календарного события.
_Avoid_: задача, дело

**Напоминание**:
Сигнал пользователю о предстоящем календарном событии или задаче с временным блоком. В MVP напоминание показывается внутри открытого приложения.
_Avoid_: push, уведомление операционной системы

**Стрим**:
Верхнеуровневая область, к которой относится работа пользователя, например работа, дом или учеба. Стрим группирует проекты и может напрямую группировать задачи без проекта; цвет стрима используется для его лейбла, а состояние определяет доступность стрима при выборе.
_Avoid_: категория, домен

**Проект**:
Более узкая область работы внутри стрима; каждый проект принадлежит ровно одному стриму. Задача с выбранным проектом относится к стриму этого проекта; цвет проекта используется для его лейбла, а состояние проекта и состояние его стрима определяют доступность проекта при выборе.
_Avoid_: стрим, список

**Состояние стрима или проекта**:
Текущая отметка стрима или проекта: активен или завершен. Проект может завершаться независимо от стрима; завершенный стрим скрывает из выбора сам стрим и все проекты внутри него.
_Avoid_: состояние задачи, архив

**Приоритет дня**:
Числовой порядок сортировки задачи внутри конкретного плана дня. Чем меньше число, тем выше задача стоит в списке задач этого дня.
_Avoid_: важность, severity

**Состояние задачи**:
Текущая отметка задачи: открыта, выполнена или отменена. Открытая задача требует внимания, выполненная завершена, отмененная больше не нужна.
_Avoid_: стадия, workflow

**Дата выполнения**:
Дата, на которую пользователь планирует выполнить задачу. Это не обязательно жесткий дедлайн.
_Avoid_: дедлайн

**Просроченная задача**:
Открытая задача, дата выполнения которой раньше сегодняшнего дня. Просроченная задача не переносится автоматически; пользователь вручную выбирает новую дату, выполняет задачу или отменяет ее.
_Avoid_: slipping, нарушенный дедлайн

**Задачи дня**:
Все открытые задачи, дата выполнения которых совпадает с выбранным днем. Задачи дня сортируются по приоритету дня.
_Avoid_: top tasks, список дел

**План дня**:
Срез задач и календарных событий, которые относятся к выбранной дате. План дня нужен для принятия решений о том, что делать сегодня.
_Avoid_: календарь, список задач

## Example Dialogue

Dev: Эта встреча из Exchange должна стать задачей?

Domain Expert: Нет, встреча остается календарным событием. Я могу создать отдельную задачу по итогам встречи, но это другая сущность.

Dev: Если позже появится второй человек в приложении, он увидит мои задачи?

Domain Expert: Нет. Задачи, проекты, стримы и подключенные календари принадлежат конкретному пользователю.

Dev: Если я подключу рабочий календарь и личный календарь, это один календарь?

Domain Expert: Нет, это два подключенных календаря. Они могут быть из разных источников, но оба отображаются в плане дня.

Dev: Чтобы посмотреть детали встречи, нужно хранить ее полное описание?

Domain Expert: Нет. В приложении достаточно минимальных данных для планирования и ссылки события, которую можно открыть в браузере.

Dev: Событие из рабочего календаря должно быть полностью залито цветом этого календаря?

Domain Expert: Нет. Цвет подключенного календаря используется как компактный маркер события.

Dev: Если ссылка на встречу указана в месте события, это место или ссылка?

Domain Expert: Это место события, из которого можно извлечь ссылку события для открытия в браузере.

Dev: Если приложение открыто на desktop, оно должно молча ждать начала встречи?

Domain Expert: Нет. Приложение должно показать напоминание о предстоящей встрече внутри интерфейса.

Dev: Если я переношу задачу с понедельника на вторник, что меняется?

Domain Expert: Меняется дата выполнения. Если у задачи был приоритет дня, его нужно пересчитать в плане вторника.

Dev: Если я перетащу задачу в слот с 14:00 до 15:00, это станет встречей?

Domain Expert: Нет, это останется задачей. У нее появится временной блок, поэтому она займет место в календарной сетке.

Dev: Если у двух задач одинаковая дата, какая будет выше?

Domain Expert: Выше будет задача с меньшим приоритетом дня. Приоритет задачи всегда читается как приоритет в рамках конкретного дня.

Dev: На сегодня открыто пять задач. Нужно показывать только три главные?

Domain Expert: Нет, задачи дня включают все открытые задачи на сегодня, отсортированные по приоритету дня.

Dev: Если задача больше не актуальна, ее надо отметить выполненной?

Domain Expert: Нет, ее надо отменить. Выполненная задача означает, что действие действительно сделано.

Dev: Если к задаче нужно приложить пару ссылок и критерии готовности, это уже заметка?

Domain Expert: Нет, это описание задачи. Заметки будут отдельным будущим функционалом.

Dev: Если открытая задача была назначена на вчера и я ее не сделал, она сама переедет на сегодня?

Domain Expert: Нет. Она станет просроченной задачей и будет показана отдельно, пока я не выберу новую дату, не выполню или не отменю ее.

Dev: Повторяющаяся зарядка каждое утро - это задача?

Domain Expert: Нет, это рутина. Обычные задачи остаются одноразовыми, а рутины отслеживаются отдельно.

Dev: Проект может быть без стрима?

Domain Expert: Нет: проект всегда принадлежит одному стриму.

Dev: А задача может быть без стрима и проекта?

Domain Expert: Да. Но если у задачи выбран существующий проект, она относится к стриму этого проекта.

Dev: Если проект завершен, он исчезает со старых задач?

Domain Expert: Нет. Завершенный проект остается на старых задачах, но не предлагается при выборе проекта для новой задачи.

Dev: Если стрим завершен, но внутри него есть активный проект, этот проект можно выбрать для новой задачи?

Domain Expert: Нет. Завершенный стрим закрывает выбор всех проектов внутри него.

Dev: Цвет проекта должен окрашивать всю задачу?

Domain Expert: Нет. Цвет используется для лейбла проекта или стрима, чтобы задача оставалась визуально спокойной.
